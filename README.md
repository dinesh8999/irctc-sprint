# IRCTC Sprint - Design Engineering & AI Feature Sprint

## Part A: Problem Discovery & Analysis

This repository documents a comprehensive audit of the Indian Railways Catering and Tourism Corporation (IRCTC) platform, identifying and analyzing critical UX, performance, and accessibility issues affecting 8 crore+ daily users.

### Assignment Overview
- **8 Crore+ daily users**
- **₹400 crore in tickets booked daily**
- **6000+ train routes**
- **Daily goal**: 12L+ tickets processed

### Part A Objectives
Study given problems → Self-discover new problems → Document with evidence → Set foundation for Part B solutions

### Repository Structure
```
irctc-sprint/
├── README.md                    # This file
├── part-a/
│   └── PROBLEMS.md             # 6 documented problems (3 given + 3 self-discovered)
├── part-b/                     # Part B deliverables (Phase 2)
│   ├── SPECS.md               # Feature specifications
│   ├── AI-FEATURE.md          # AI solution proposal
│   └── MATRIX.md              # Impact prioritization matrix
└── assets/
    └── screenshots/           # Evidence from live platform exploration
```

### Problems Documented
- **3 Given Problems**: Tatkal crashes, Search filter failures, Seat selection resets
- **3 Self-Discovered Problems**: Discovered through live platform exploration

See `part-a/PROBLEMS.md` for complete documentation.

## How This Was Created
All problems were discovered by exploring the live IRCTC platform at **irctc.co.in** and documented with:
- Step-by-step broken user flows
- Specific affected user segments
- Frequency analysis from direct observation
- Evidence screenshots from the actual platform
- Technical root cause analysis
